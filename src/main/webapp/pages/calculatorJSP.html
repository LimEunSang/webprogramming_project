웹프로그래밍 프로젝트

2022.05.27

JSP 계산기의 동작 구조는 이렇습니다. 식을 입력하고 연산 버튼(=)을 누르면 입력값을 JSP로 전달합니다. JSP는 브라우저로부터 전달된 입력값을 가져와 계산 후 결과를 포함한 화면을 출력합니다.

html 파일 코드와 jsp 파일 코드 참고해주세요.

html 파일에서 서블릿으로 구현할 때와 다른 점은 form 태그의 action 속성이 호출할 jsp 파일의 상대 경로라는 것 딱 하나입니다. 나머지는 모두 같습니다. 따라서 UI도 똑같습니다.

jsp 파일에 서블릿으로 작성한 java 파일의 doGet() 메서드 내용을 8~20행과 같이 스크립트릿 영역에 넣어줍니다. html 부분에서 결과를 출력해야 하므로 스크립트릿 영역 코드는 앞쪽에 배치합니다. 4~6행 코드는 jsp에서 import 하는 명령어입니다. 서블릿에서 구현할 때와 같이 javascript 코드를 사용하기 위해 삽입한 import문 입니다.

JSP를 사용하면 java 코드로 작성한 객체를 31행과 같이 html에서 쓸 수 있는 장점이 있습니다. 따라서 동적 웹페이지를 구성할 수 있죠. 마치 javascript를 `${객체}` 형식으로 사용하는 것 같이요.

java 코드 영역에서 동적으로 출력해 줄 객체를 정의했으면 그다음엔 22~33행과 같이 일반 html 파일 작성하듯 웹페이지를 만들면 됩니다. 딱 봐도 서블릿에 비해 상당히 효율적이네요.

서블릿 계산기 구현 페이지에서도 말했지만, 이는 상당히 비효율적인 구현입니다. 서버를 통한 JSP의 동작 원리만 이해하고 넘어갑시다😘
